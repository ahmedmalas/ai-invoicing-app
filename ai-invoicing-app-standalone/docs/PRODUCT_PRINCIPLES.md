# AI BUSINESS OS - PRODUCT PRINCIPLES

## Mission

Build software that remembers, learns, automates and assists so people spend less time on administration and more time growing their business and enjoying their lives.

## Core Principles

1. The platform adapts to the user.
   The user never adapts to the software.

2. If the user performs the same task twice, first ask: "Why didn't the software remember it?"

3. Everything should be searchable.
   Documents.
   Customers.
   Suppliers.
   Photos.
   Receipts.
   Invoices.
   Quotes.
   Payments.
   Notes.
   AI memories.
   Everything.

4. Nothing should exist in isolation.
   Every document should link to:
   Customer
   Supplier
   Payments
   Timeline
   Photos
   Notes
   Documents
   Jobs
   History

5. Every action creates history.
   Nothing disappears.
   Everything has a timeline.

6. AI is an assistant.
   Never replace user decisions.
   Recommend.
   Explain.
   Learn.
   Remember.
   Ask.
   Never silently change important financial information.

7. Every major feature must save time.
   If it doesn't save time, question whether it belongs.

8. The platform is document-first.
   Invoices are only one document type.
   Everything is built on a shared document engine.

9. The platform works perfectly without integrations.
   Integrations enhance it.
   They never become mandatory.

10. User ownership.
    The user's data belongs to them.
    Easy export.
    Easy backup.
    Easy migration.
    No lock-in.

11. Professional by default.
    Every generated document should be suitable for sending to a customer immediately.

12. Everything should feel connected.
    Customer profiles.
    Supplier profiles.
    Payments.
    Expenses.
    Documents.
    Branding.
    AI memory.
    Accounting.
    Everything is one ecosystem.

13. Personal and Business editions share one platform.
    Only capabilities change.
    Never create two completely different products.

14. Security first.
    Audit logs.
    Permissions.
    Encryption.
    Approval workflows.
    Privacy.

15. AI Memory
    The platform continuously learns user preferences with permission.
    Brand.
    Templates.
    Writing style.
    Customers.
    Suppliers.
    Layouts.
    Reminder schedules.
    Favourite actions.
    Recurring behaviour.

16. Simple first.
    Powerful second.
    Never overwhelm a new user.
    Advanced features should unlock naturally.

17. No feature should be added unless it clearly supports the mission.

## Things We Will Never Become

- Bloated accounting software.
- Software requiring ten screens to create one invoice.
- Software that forces users into one workflow.
- Software that locks users into one ecosystem.
- Software that forgets user preferences.
- Software that hides important information.

## Future Design Philosophy

Every future feature proposal must answer:

1. Does it save the user time?
2. Does it reduce manual work?
3. Does it fit the AI Business OS vision?
4. Will it still make sense five years from now?

If the answer is no, it should probably not be built.
