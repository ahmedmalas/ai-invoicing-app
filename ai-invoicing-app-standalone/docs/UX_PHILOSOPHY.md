# UX Philosophy: Progressive Complexity

## Principle
Simple first. Powerful second.

## Requirements
- New users should never feel overwhelmed.
- Simple workflows should stay simple and fast.
- Advanced workflows should unlock naturally as user needs mature.
- Power users should retain deep control without forcing complexity on everyone.

## Product Behavior Implications
- Default paths should minimize decisions for common tasks.
- Advanced options should be discoverable, not intrusive.
- Repeated behavior should become easier through AI memory and sensible defaults.
